# Features

This folder is for future features that are concrete enough to deserve implementation-oriented documentation before code exists.

Use it when a feature needs one or more of:

- a scoped problem statement
- implementation planning
- references to external projects or docs
- open questions and tradeoffs
- staged rollout notes

Suggested structure:

- one folder per feature
- `README.md` for feature overview
- `implementation-plan.md` for the execution plan
- `resources.md` for source material, references, and notes

This keeps rough brainstorming in `docs/ideas/` and feature-grade planning here.


Status conventions:
- `Implemented`
- `Partial`
- `Planned`
- `Archived`
