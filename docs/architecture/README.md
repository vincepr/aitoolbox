# Architecture

This folder holds the current architectural intent for the repository.

Initial focus:

- define stable boundaries before building tools
- keep the core domain-agnostic
- make execution strategy a first-class concern
- separate human-authored definitions from operational state
- favor inspectable and validated configuration
